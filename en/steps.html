Import projects

When you click Import, 2 entities will be created in your workspace: the SimpleApp application, conatining three message flows, and the SimpleBar BAR file.

The contents of this tutorial will be explained in more detail when you move on to the Prepare tab, along with the setup steps you need to complete before deploying to a server.

Import

Prepare an Integration Server

  1. Assuming you are using a Windows platform, open an App Connect Enterprise Command Console and start a standalone integration server using the following command:
    integrationserver --name server0 --work-dir C:\server0 -admin-rest-api 7600 --http-port-number 7800 --console-log
  2. Now we will create an Integration Node and launch some servers to it. Open a new ACE Command Console and type:
    mqsicreatebroker myNode -P 4414
    mqsistart myNode
    mqsicreateexecutiongroup myNode -e nodeServer0
    Repeat the final command and replace the 'nodeServer0' to create a 'nodeServer1', 'nodeServer2' and 'nodeServer3'.
  3. Then, open a third ACE Command Console and set the environment variable ACE_DASHBOARD=true by typing:
    setx ACE_DASHBOARD true
  4. Finally, start a new integration server in this ACE Console by typing the command:
    integrationserver --name serverDash --work-dir C:\serverDash -admin-rest-api 7601 --http-port-number 7801 --console-log
  5. Open a web browser connect to the following 3 locations in seperate tabs with:
    localhost:7600
    localhost:4414
    localhost:7601

Deploy

Follow these steps to complete the tutorial

Follow the following steps to complete the tutorial.

  1. In your browser open the tab connected to localhost:7600, this is the WebUI for your standalone integration server. You should see 5 tabs along the top of the site, a Deploy button on the right and an empty body of Contents.
  2. Now we'll deploy our SimpleBar to this server using the WebUI. Click the 'Deploy' button on the right of the page. Click on 'Select a BAR file' and navigate to your toolkit's workspace. Open the SimpleBar.bar under the 'BARfiles' directory and deploy it.
  3. SimpleApp should appear under the 'Contents' tab on the WebUI. Click on it to view the contents of the app, in this case just 3 simple flows should appear. Clicking on each of the flows will display the properties and statistics of that flow.
  4. Explore the other 4 tabs as you wish to gather information about this specific server. We will explore Policies and how to utilise both Statistics tabs in later tutorials.
  5. Next, open the localhost:4414 tab and you should see all the servers you created under your Integration Node earlier. Enter each of them and follow the above steps to deploy SimpleBar.bar to as many as you like. Here you can manage many integration servers from one UI.
  6. Return back to the server view in your Node WebUI. On the right there should be a button labelled 'Create a Server'. Click it and name your new server something unique. Deploy the Bar file to it if you like.
  7. Open the localhost:7601 tab. In the top right of this web UI there should be 3 brand-new tabs and an 'Edit Connections' button.
    - Click on 'Edit Connections'
    - Under the Connections tab, click 'Add'
    - Select 'Integration Server', name it 'server0' and set the address to 'http://localhost:7600'
    - Click 'Save' in the bottom right of the window, you should now be able to access your standalone server directly from your Dashboard's WebUI.
  8. Repeat the above steps to connect to your Integration Node at the address 'http://localhost:4414'.
  9. Now you will be to manage all your Integration Servers and Nodes from one handy Web User Interface. Changing the focus to 'Integrations' in the top right will shwo all the deployed applications across all servers you own. 'Nodes' and 'Servers' display all nodes and servers respectively.
